# LPM 1.0: Video-based Character Performance Model

# Abstract
表演是意图、情感和个性的外部体现，通过视觉、听觉和时间行为来传达，它是赋予角色生命力的核心 。从视频中学习这种表演是替代传统 3D 制作流程的一条极具前景的路径 。然而，现有的视频模型难以同时实现高表现力、实时推理和长时特征稳定性，这种紧张关系我们称之为“表演三难困境” (Performance Trilemma) 。

对话是最全面的表演场景，因为角色在保持身份一致性的同时，需要同步进行说话、倾听、反应和情感表达 。为解决这一问题，我们推出了 LPM 1.0，专注于单人全双工音视频对话表演 。具体而言，我们的工作包括：
* 多模态数据集构建：通过严格筛选、说听音视频配对、表演理解以及身份感知多参考提取，构建了一个以人为中心的大规模数据集 。
* Base LPM 训练：训练了一个拥有 170 亿 (17B) 参数的扩散变换器 (Diffusion Transformer)，通过多模态条件控制实现高度可控且身份一致的表演 。
* Online LPM 蒸馏：将其蒸馏为一个因果流式生成器，用于低延迟、无限长度的实时交互 。

在推理阶段，给定角色图像及身份感知参考资料，LPM 1.0 能根据用户音频生成倾听视频，根据合成音频生成说话视频，并结合文本提示进行动作控制 。所有生成过程均达到实时速度，且具备长时身份稳定性 。LPM 1.0 可作为对话智能体、直播虚拟人及游戏 NPC 的视觉引擎 。此外，我们提出了 LPM-Bench，这是首个针对交互式角色表演的基准测试 。实验证明，LPM 1.0 在所有评估维度上均达到了 SOTA（先进水平），并保持了实时推理能力 。

# 1. Introduction
“表演就是反应。” —— 桑福德·迈斯纳 (SANFORD MEISNER) 

表演从根本上说是具有反应性的：它产生于社会互动中行为的持续、具身协作 。表演不仅通过言语传达，还通过节奏、呼吸、犹豫、停顿、凝视、姿势、手势以及无数揭示注意力、意图和情感的微表情来体现 。让交互显得自然的关键在于时刻做出反应的能力：在开口前倾听、预测轮换时机，并随着情况的发展来缓和、强化或保留回应 。因此，一个表演模型要做的不应仅仅是让语言动起来 。给定角色设定和时变的多模态控制信号，它必须维持一个能够随着时间推移进行说话、倾听、反应、表达情感和移动的角色 。在这项工作中，我们致力于为交互式对话中的虚拟角色带来这种表演能力 。

如今，电影级质量的角色表演仍主要通过传统的 3D 流程交付，涉及建模、绑定、动画和渲染 。尽管这些流程因其显式的模块化和可控性而功能强大，但它们本身难以扩展到我们所针对的开放式设定中 。扩展覆盖范围——涵盖新角色、新行为（如不同表演风格、情感和交互背景）以及多模态（如音频、面部表情、身体运动和视觉外观之间的忠实对应）——通常需要资产、采集数据和专家人力的成倍增加 。随着交互式应用的激增——从类人智能体到大规模开放世界的对话式 NPC——核心挑战不再是打造单一的高端表演，而是为众多角色在各种场景下快速、可靠且以极低的人工创作成本生成多样化的表演 。最近一些基于 3D 的研究开始探索交互式对话场景，但它们仍继承了 3D 流程的根本局限性 。

这促使我们选择了一条截然不同的路线：直接在像素空间中从视频学习角色表演 。大规模视频数据天然包含身份、视角、对话背景、表达情感和表演风格的广泛变化，这意味着单一的学习模型可以摊销不同角色和场景的创作成本 。同时，基于扩散的视频生成技术的最新进展显著提升了视觉保真度，而音频和文本则为交互式使用提供了天然且低门槛的控制信号 。这些进展共同使得从参考图像、音频和文本中合成角色行为成为可能，而无需重新制作角色特定的绑定或采集流程 。

然而，现有的视频生成模型面临着一个极其僵化的约束，我们将其称为表演三难困境 (Performance Trilemma) 。在实践中，现有模型难以同时满足以下三个关键要求 ：表现力质量：具备像真人一样表演的能力，展示丰富、对话化且非重复的动作、微行为以及交流性的凝视 。实时推理：具备适合直播的因果、实时生成能力 。长时稳定性：在无限长的时间内保持身份、解剖结构、个性化风格和视觉保真度 。

优化速度和稳定性通常会导致机械、重复的动作；优先考虑表现力通常会破坏实时响应能力；而要在长时跨度内保持身份和保真度，对于自回归漂移来说仍是一个挑战 。因此，现有系统是称职的“说话者”，却是糟糕的“表演者” 。

对话同时对这三个要求施加了压力：表现力涵盖了从积极说话到专注倾听的范畴；推理必须是实时的以支持轮换和反应行为；且身份在无限的交互过程中必须保持稳定 。我们将对话表演的局限性归因于当前范式中缺失的四个要素：
1. 倾听行为基本缺失：大多数模型排他性地由语音驱动，忽视了倾听者的关键角色——包括专注的凝视、个性化的反应、预测以及轮换信号 。这些元素构成了自然对话的一半，对于传达理解感和存在感至关重要 。
2. 多模态可控性仍然薄弱：真正的表演需要对时变的文本指令（用于指导面部表情、身体动作和环境互动）、说话音频（驱动口型同步、情感和节奏感动作）以及倾听音频（触发反应性和理解性行为）进行联合建模 。
3. 角色设定描述不足：仅依赖单张参考图像作为条件，迫使生成模型不得不去凭空推测未见过的细节，如牙齿、表情皱纹、侧面几何结构以及新视角下的身体外观 。虽然这些推测在逐帧看来似乎合理，但往往难以在长序列中保持一致 。这种欠规格化使得单图条件不足以满足需要稳定身份和动作连续性的生产场景 。
4. 缺乏以人为中心的视频基础模型：现有模型通常构建在小规模或特定领域的训练集上，未能从大规模、高质量的以人为中心的视频预训练中获益 。这种基础模型的缺失导致视觉保真度有限、在多样化身份和场景下的泛化能力弱、多模态可控性差以及对话能力不足，阻碍了这些系统作为通用表演引擎的功能 。

为了解决表演三难困境，LPM 1.0 提供了一个用于视频生成式对话表演的全栈框架，涵盖了高质量数据构建、具备多模态条件的基座模型训练，以及用于实时部署的在线蒸馏 。我们认为表演三难困境不仅是架构上的挑战，更是系统层面的挑战 。解决它需要跨数据、控制和推理的协同设计：数据必须涵盖广泛的说话、倾听和社交反应行为；多模态调节必须使这种行为空间在推理时可控；且部署必须在因果、低延迟的流式约束下保留这些能力 。我们通过以下三个核心要素实例化了该框架：
(1) 数据流程：我们通过三个专用管线构建了大规模多模态数据集：严格的质量过滤、对话音视频配对数据处理以及身份感知参考图像构建 。这些管线产出的训练数据不仅视觉清晰，而且明确适用于全双工对话生成，包括说话和倾听行为以及丰富的身份规格描述 。
(2) Base LPM：基于此数据集和拥有 140 亿参数的预训练图生视频基础模型，我们以交替说话/倾听音频交叉注意块的形式引入了 30 亿额外参数，得到了一个 170 亿参数的模型，并在数千万个剪辑上进行了全面训练 。这构成了 Base LPM，它联合学习了四项核心能力：语音驱动的时间动力学、内容感知倾听反应、细粒度文本调节控制以及多参考身份保留 。Base LPM 能生成可控、高质量且具有分钟级时间相干性的角色表演视频 。
(3) Online LPM：我们将 Base LPM 蒸馏为 Online LPM，这是一个用于低延迟、无限长度交互的因果流式生成器 。我们开发了一种四阶段自回归蒸馏课程，逐步将基座模型的视觉质量和行为丰富度转移到实时“骨干-精炼器”(backbone-refiner) 架构中：骨干网络维持具有时间相干性的潜空间轨迹，而精炼器负责恢复高保真细节 。这种分离使 Online LPM 能够在实际延迟约束下，在无界的时间跨度内保持身份、同步性和动作真实感 。

LPM 1.0 因此可作为对话智能体、直播虚拟角色和游戏 NPC 的视觉引擎 。为了评估这一新领域，我们引入了 LPM-Bench，这是首个专为具备多模态输入的交互式角色表演设计的基准测试 。

我们的贡献如下：
我们提出了 LPM 1.0，这是首个用于单人全双工对话表演的视频生成系统，提供了一个共同解决表现力、实时推理和长时稳定性的全栈框架 。
我们通过严格的流程构建了大规模、精选的多模态数据基础，实现了大规模学习表现力和反应性的角色行为 。
我们开发了 Base LPM，一个在超过 1.7 万亿多模态 token 上训练的 17B 双向扩散变换器，它联合建模了语音驱动动作、倾听行为、文本引导控制和多参考调节 。
我们引入了多阶段自回归蒸馏框架，将 Base LPM 转换为 Online LPM，实现了实时、无限长度的合成 。
我们构建了 LPM-Bench，并证明 LPM 1.0 在对话自然度、可控性、情感表演和长期一致性方面均达到了 SOTA 结果 。

# 2. 数据 
训练一个用于交互式视频生成的表演模型，所需的数据远超传统的说话人脸（talking-head）语料库 。这样的数据集必须满足三个关键要求 。首先，它必须提供以人为中心的高质量视频，具备广泛的视觉和动作覆盖，涵盖多样的镜头景别、动作、表情、情感状态和外观属性，以便模型能够泛化到现实世界设置中遇到的各种人类视觉呈现 。

其次，它必须捕捉丰富的对话表演，包括说话和倾听行为、多轮交互动力学以及反应性的社交信号，同时涵盖塑造对话的社交因素，如人设（persona）、关系类型、性格、话题、情感和情境背景 。这种多样性对于学习不仅视觉上合理，而且在社交层面上有理有据且符合上下文逻辑的行为至关重要 。

第三，它必须支持从原始视频中提取的成对多模态条件，包括分为说话和倾听流且与帧对齐的音频、用于细粒度动作控制的详细描述，以及多粒度的身份参考图像，以在长时生成中强制保持角色身份的一致性和表情动力学的保真度 。

因此，我们通过对多样的长视频进行多阶段处理管线，构建了一个专门针对对话导向生成建模的可扩展、以人为中心的多模态数据集 。与以往主要强调纯说话视频生成的规模化尝试（如 OpenHumanVid 和 SpeakerVid-5M）相比，我们的管线目标是在野外场景下的高质量对话表演，必须共同处理对话语义、详细文本和身份一致的调节条件 。通过严格的过滤和分布感知平衡，总保留率控制在 10% 以下，从而产出具备高质量、多样性和标注密度的训练剪辑，满足高保真对话角色生成的需要 。

## 2.1. 准备以人为中心的高质量视频 
数据采集。我们策划了一个旨在平衡多样性、覆盖范围和质量的大规模视频语料库 。收集的数据涵盖了广泛的表演场景，在镜头尺度、身体动作、面部表情、情感状态和外观属性方面具有显著差异 。为了更好地捕捉全双工对话行为，该语料库还包括了具有清晰可辨的发言者-听者互动的对话驱动录像，涵盖了不同的人设、人际动力学和情感丰富的场景 。所有数据都由庞大的真人标注团队进行系统的质量和内容审计，在进入处理管线之前核实其视觉保真度、内容相关性和对话有效性 。

数据过滤与分类。整个过滤和分类过程如图 2 所示 。具体而言，我们通过多阶段清洗管线将异构的原始视频转换为标准化的、高质量的单镜头剪辑 。原始视频首先通过场景检测被分割成时间相干的单镜头，随后使用人体检测模型去除没有可见人物或人群密度过高的剪辑 。

为了提升音视频质量，我们采用人工检查与基于模型的方法相结合，过滤掉五个主要类别的有缺陷数据 ：视频编辑和后期处理痕迹：如转场、跳切、丢帧、后期叠加物（Logo、字幕、水印、特效）以及美颜/磨皮滤镜 ；视觉质量缺陷：包括失焦画面、全局偏色和剧烈的屏幕闪烁 ；内容真实性问题：如静态图像素材、AI 生成的视频或具有明显合成/绿幕背景的场景 ；构图问题：包括不完整的人物形象以及双屏或画中画布局 ；音视频同步问题：涵盖音画不同步以及人物嘴部有动作但无声音的视频 。

通过该管线，在最终人工质量抽检中识别出的缺陷数据比例保持在 1% 以下 。最后，保留的剪辑按可见人数进行分类，生成结构化元数据 。输出结果是一个高质量、时间相干、标注了人数的单镜头剪辑语料库，可用于后续的语义标注 。

## 2.2. 对话音视频数据处理 
对话视频描绘了角色参与多轮视听对话、在说话和倾听之间轮换的场景 。为了生成真实的对话行为，模型需要逐帧、逐角色的条件，明确标注每个可见人物在每一时刻的状态（说话、倾听或闲置），不仅要捕捉语言产出，还要捕捉体现自然对话特征的非言语反应，如点头、微笑、改变凝视方向或转移注意力 。大规模获取此类标签面临若干挑战 ：
1. 有限观测下的对话推理：当只看到交流的一方时，判断一段简短剪辑是否属于对话并非易事 。这通常需要多模态推理：语音内容可能暗示对话，而视觉上下文提供辅助证据（例如画幅边缘有局部可见的交谈者） ；
2. 定义广泛行为谱系下的“倾听”：在对话中的倾听表现为专注的回应（如点头、微笑），这些行为与对话伙伴的语音紧密耦合 。另一方面，通用的音频反应行为包括由非语音音频驱动的响应（如随音乐摇摆）。其统一准则是可见人物的行为是否与并发的音频输入表现出有意义的对应关系 ；
3. 幼稚启发式方法的脆弱性：仅基于背景语音将帧标记为“倾听”常导致误报，因为画外旁白或无关噪声缺乏参与感的视觉证据 。

具体而言，帧级状态分类依赖于两个基础信号：用于识别语音何时出现及其所属身份的语音检测，以及用于确定可见人物嘴唇活动是否与检测到的语音对应的音视频同步性 。分类逻辑如下 ：
* 在检测到的语音段内：嘴唇动作与音频同步的人被分类为“说话”；嘴唇保持不动的人被分类为“倾听”；嘴唇有动作但与音频不同步的人被标记为不一致数据并丢弃 。
* 在语音段之外：嘴唇不动的人被分类为“闲置”（idle）；嘴唇有不明动作的人被丢弃 。

我们的管线如图 3 所示。我们同时保留单人及多人剪辑；多人镜头通常包含更丰富的对话动态（如轮替和相互关注），因此能为对话导向的建模产出比例更高的有用训练样本。因此，在对话策划过程中，我们强调多人剪辑，同时保留单人剪辑作为补充覆盖。

无论人数多少，每段剪辑首先都要经过行人检测和跟踪，以获取时间一致的边界框轨迹。对于单人剪辑，全画幅即构成唯一的人物中心轨道；对于多人剪辑，则利用轨迹裁剪出独立的人物中心视图，并通过质量过滤剔除低保真轨道。

随后，我们对每条人物中心轨道独立进行帧级音频状态标注，生成各人状态预测。对于多人剪辑，角色分配与音频分离阶段会解析说话者-听者角色，并构建基于角色的音频变体；随后将剪辑分解为单人片段，每个片段与对应的音频变体配对。分解后，我们将这些剪辑输入音视频理解模型进行对话场景分类。

帧级音频状态标注：我们通过迁移学习，将 ASD（活跃说话者检测）主干网络（LR-ASD）微调为三状态帧级表述。针对训练集，我们标注了一个包含 2 万个单镜头剪辑（约 95 小时）的内部数据集，显著提升了在标准 ASD 设定之外区分“倾听”与“闲置”的能力。适配后的模型联合利用语音检测和音视频同步性，实现上述分类逻辑：将每一帧标注为说话、倾听或闲置，同时标记并移除音视频不一致的帧。

角色-音频分离与分配：我们引入了一个规则驱动的帧级重排序模块，在时间上聚合各人概率并实施一致性约束。具体流程包括：(1) 逐帧对比候选轨道的说话/倾听置信度；(2) 应用时间平滑和持久性检查以抑制瞬时尖峰；(3) 通过偏向于跨轨道的稳定说话人模式和连贯的轮替转换来解析角色分配。基于解析结果，我们为每个多人剪辑构建两种音频变体：通过抑制分配给听者的语音获得“仅说话者轨道”，以及通过抑制分配给说话者的语音获得“仅听者轨道”；闲置区域在两条轨道中均保留。
