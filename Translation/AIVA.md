# title
AIVA: An AI-based Virtual Companion for Emotion-aware Interaction

# abstract
近年来，大语言模型（LLMs） 的显著进步极大地提升了自然语言的理解与生成能力，从而增强了人机交互（HCI） 。然而，大语言模型目前仍局限于单模态文本处理，缺乏从非语言信号中解读情感线索的能力，这阻碍了更具沉浸感和共情力的交互体验 。本研究探索将多模态情感感知集成到大语言模型中，以创建具备情感意识的智能体 。我们提出了 AIVA——一个基于人工智能的虚拟伴侣，它能够捕捉多模态情感线索，实现情感对齐且生动的人机交互 。AIVA 引入了一个多模态情感感知网络（MSPN），该网络利用跨模态融合 Transformer 和有监督对比学习来提供情感线索 。此外，我们开发了一种情感感知提示工程（EPE）策略用于生成共情响应，并集成了文本转语音（TTS）系统和动画虚拟形象模块，以实现更具表现力的交互 。AIVA 为情感意识智能体提供了一个框架，在伴侣机器人、社交护理、心理健康和以人为中心的人工智能领域具有广泛的应用前景 。

# 1 introduction

近年来，大语言模型（LLMs）在自然语言理解和生成方面取得了显著进展 。通过在海量文本数据上进行数以十亿计参数的训练，ChatGPT、LLaMA 和 ChatGLM 等大语言模型在各类语言任务中展现出了惊人的能力，包括问答、常识推理和开放式对话 。凭借其强大的泛化能力和上下文学习能力，LLMs 已显示出作为智能体和虚拟助手核心引擎的巨大潜力，使广泛现实应用场景中的人机交互变得更加自然、灵活且人性化 。

然而，LLMs 往往难以准确捕捉用户的心理状态，因为情感线索并不局限于文本输入，而是广泛地通过非语言模态表达，例如面部表情、语音语调和肢体动作 。例如，“没关系（It's okay）”这样一个简单的句子，根据用户的面部表情或语调不同，可能表达完全不同的情感——从真诚的接受到深度的沮丧或悲伤 。

这一局限性阻碍了 LLMs 在共情共鸣型人机交互（HCI）领域的有效应用 。一个理想的人机交互系统（如图 1 所示）应当通过多模态线索感知用户的意图和情感状态，结合语言、面部表情和声音来支持情感智能交互 。

多模态情感计算旨在通过分析来自多个模态的信号来理解和建模用户的情感状态 。该领域的近期进展催生了多种架构，这些架构集成了视觉、听觉和文本信息，用于情感分类、情感识别和用户状态跟踪 。这些系统在检测各种场景下的细粒度情感状态方面表现出了强大的性能 。

然而，现有的方法通常与大语言模型（LLMs）孤立开发，限制了它们在交互式对话系统中的适用性 。鉴于 LLMs 强大的上下文学习能力，将外部多模态情感感知模块整合进由 LLM 驱动的智能体中具有巨大潜力，这将使其能够根据实时情感线索调整响应内容 。

# 2 related work
## 大语言模型 (LLMs)
近年来，大语言模型（LLMs）取得了显著进展，在语言理解、常识推理和问答方面表现卓越 。这些模型基于 Transformer 架构，通常包含数十亿个参数，并在海量的互联网数据集上进行预训练以获取世界知识 。知名模型如 BERT、GPT 和 T5，各自在架构和预训练方法上有所不同 。BERT 使用掩码语言模型进行上下文学习，而 GPT 采用自回归训练以提升语言生成能力 。T5 则集成了编码器-解码器架构，通过统一的模型处理多样化的 NLP 任务 。更近期的模型如 LLaMA 提升了训练效率和性能，而如 Alpaca 所示的指令微调方法则增强了特定任务的理解能力 。GLM 通过结合生成式预训练和掩码语言建模进一步改进了 LLMs，在语言理解和生成方面均表现出更强的性能 。尽管 LLMs 在文本任务中取得了成功，但在整合非语言线索方面仍面临挑战，这使得它们在需要视觉和语音等模态信号的**人机交互（HRI）**中效果较差 。

## 多模态学习
多模态学习通过集成来自多个模态（如文本、图像、音频）的数据，提供全面的证据来改进决策过程 。这种方法利用多样化的数据源增强了神经网络在跨模态任务中的表现 。多模态学习的近期进展包括 CLIP 和 ViLT，它们利用对比学习和 Transformer 来对齐视觉和文本表示 。BLIP 通过为分类和生成等跨模态任务提供统一的方法扩展了这一领域 。更近期的模型，如 BLIP-2、LLaVA 和 MiniGPT-4，弥合了视觉与语言之间的鸿沟，使 LLMs 能够根据视觉输入生成文本 。这些模型采用了创新的策略，如可学习的投影层和跨注意力机制，以增强跨模态对齐并提高在多模态场景下的推理能力 。

# 3 method
1. AIVA 框架概述 AIVA 框架旨在将多模态情感感知与大语言模型（LLMs）相结合，构建一个情感智能虚拟伴侣，从而实现共情化的人机交互（HCI）。如图 2 所示，AIVA 通过多模态情感感知网络（MSPN）从文本和视觉输入中捕捉用户的情感线索 。MSPN 融合这些模态提取出的情感信号，随后通过情感感知提示工程（EPE）引导预训练的大语言模型生成响应 。该框架在生成上下文适配且富有共情力的语言回答时，还通过文本转语音（TTS）系统和动画虚拟形象模块提供富有表现力的反馈 。
1. 提出的 MSPN MSPN 旨在检测用户的实时情感状态并为 LLMs 生成情感线索 。
   * 编码器：对于图像-文本对，使用 Vision Transformer (ViT) 作为视觉编码器提取视觉表示，同时使用 BERT 作为文本编码器提取文本表示 。
   * 跨模态融合 (CAF)：为了实现有效的模态间交互，MSPN 采用了跨注意力（Cross-Attention）机制 。每一模态的表示作为查询（Query），而另一模态则作为键（Key）和值（Value）。
   * 融合 Transformer (CMFT)：统一的多模态表示会被送入跨模态融合 Transformer，以实现深层融合 。此处引入了一组可学习的情感原型（Sentiment Prototypes），通过跨注意力机制从融合特征中捕捉特定的情感信息 。
   * 情感识别：最终层的输出被送入多层感知机（MLP）分类器进行情感分类 。
1. 训练目标 为了促进有效的表示学习，MSPN 采用了分类损失和**有监督对比学习（SCL）**目标 ：
   * 分类损失：使用交叉熵损失根据真实标签预测情感 。
   * 有监督对比损失：将多模态表示与相应的情感原型进行对齐 。
最终的训练损失是分类损失与双向对比损失的加权之和 。
4. 情感感知提示工程 (EPE) 为了将 MSPN 捕捉的情感线索注入 LLM，我们采用了情感感知提示工程 。
   * 该方法将情感分类结果作为 Prompt 的前缀，引导 LLM 生成情感对齐的响应 。
   * Prompt 模板包含：角色定义（Role）、少样本示例（Few-shot）、历史上下文（History）以及**思维链（CoT）**引导，使模型在生成响应前进行逐步推理 。
1. 文本转语音与动画虚拟形象 TTS 系统：由 GPT-SOVITS 驱动，将文本转换为自然且富有情感表现力的语音 。
   * 动画形象：使用 Live2D 创建，提供动态的面部表情和肢体动作，增强交互的情感投入感 。

# 4 experiment
1. 实验设置
* 数据集：我们在多个多模态情感分析数据集上评估了提出的 MSPN 。
  * MVSA-Single：一个包含图像和文本的基准数据集，标注为积极、中性或消极，用于情感分类微调 。
  * MVSA-Multiple：扩展了多标签情感标注，用于评估模型处理跨模态微妙情感状态的能力 。
  * TumEmo：包含超过 195,000 个“图像-文本-情感”三元组，用于 MSPN 的预训练以学习跨模态情感信号 。
* 基准模型：我们将 MSPN 与 OSDA、MultiSentNet、HSAN、Co-MN-Hop6 和 MGNNS 等模型进行了对比 。对于 AIVA 系统，我们选择 LLaMA2-Chat 作为预训练的大语言模型 。
* 超参数设置：学习率设为 $2\times10^{-5}$ 。MVSA 数据集的训练批次大小（batch size）为 16，TumEmo 预训练则为 24 。使用 Adam 优化器 。
1. 实验结果
   * MVSA 对比结果：表 1 显示 MSPN 的性能优于所有基准模型，在 MVSA-Single 上达到了 74.25% 的准确率和 72.84% 的 F1 分数 。结果表明，MSPN 有效结合了多模态感知，显著提升了分类性能 。
   * TumEmo 预训练结果：MSPN 在 TumEmo 数据集上达到了 81.81% 的准确率和 81.89% 的 F1 分数 。
   * 可视化分析：图 6 的 t-SNE 可视化显示，模型能有效区分“快乐”、“悲伤”和“愤怒”等情感 。虽然“恐惧”和“悲伤”之间存在一些重叠，但整体分类效果良好 。
1. 消融研究MSPN 组件分析：表 3 的结果验证了各个组件的有效性 。**跨注意力融合（CAF）**对融合质量至关重要，**跨模态融合 Transformer（CMFT）增强了特征对齐，而有监督对比学习（SCL）**确保了情感分类的鲁棒性 。
* 超参数 $\lambda$ 的敏感性：表 4 显示当 $\lambda = 1.0$ 时模型表现最佳 。过高或过低的 $\lambda$ 值都会导致性能下降，说明分类损失与对比损失之间的平衡至关重要 。
1. 定性分析MSPN 案例：模型能准确预测情感，例如将冬季景观与“快乐”联系起来，或从哭泣的角色图像中识别出“悲伤” 。AIVA 交互案例：如图 8 所示，当用户分享获得新宠物的兴奋时，AIVA 会以愉快的文字和开心的虚拟形象进行回应；当用户因手机摔坏而沮丧时，AIVA 则提供同情的回应并切换为悲伤的情感形象 。

# 5 ablation study
1. MSPN 的消融分析在此部分，我们进行了消融实验，以验证 MSPN 中不同组件的有效性，包括跨注意力融合（CAF）、跨模态融合 Transformer（CMFT）以及有监督对比学习（SCL） 。表 3 的结果显示，移除任何一个组件都会导致性能下降，这突显了每个部分的必要性 。
具体而言：
* CAF 在提升融合质量方面起着至关重要的作用 。
* CMFT 增强了多模态特征的对齐 。
* SCL 确保了情感分类的鲁棒性 。完整的 MSPN 模型性能最佳，优于所有消融变体 。
1. 对参数 $\lambda$ 敏感性的评估表 4 展示了针对超参数 $\lambda$ 的消融研究结果 。结果表明，当 $\lambda = 1.0$ 时，模型表现最优，在 MVSA-Single 数据集上达到了 74.25% 的准确率和 72.84% 的 F1 分数，在 MVSA-Multi 数据集上达到了 73.48% 的准确率和 70.01% 的 F1 分数 。
* 较低的 $\lambda$ 值（如 0.5）会导致性能下降 。
* 较高的 $\lambda$ 值（如 1.5 和 2.0）同样会引起性能下滑，尤其是在 F1 分数方面 。
* 这表明在分类损失与对比损失之间取得适当的平衡对于获得最佳性能至关重要 
# 6 conclusion
1. 多模态情感感知网络 (MSPN) 
* 不同于传统的单模态模型，AIVA 引入了 MSPN，通过 Cross-modal Fusion Transformer（跨模态融合 Transformer） 深度整合视觉和文本信号 。
* 该网络利用可学习的情感原型（Sentiment Prototypes）和有监督对比学习（SCL），使模型能够在特征空间中精准区分细微的情感差异 。
2. 情感感知提示工程 (EPE) 
* AIVA 并没有直接修改 LLM 的底层架构，而是通过 EPE 将 MSPN 提取的情感线索作为前缀注入 Prompt 。
* 结合了角色定义、少样本学习（Few-shot）和思维链（CoT），引导 LLaMA2 生成在逻辑和情感上都与用户同步的共情响应 。
3. 多感官闭环交互体验 
* 系统集成了 GPT-SOVITS（情感语音合成） 和 Live2D（动态看板娘），实现了从情感识别到富有表现力的声音、画面反馈的完整闭环 。
* 这种设计显著提升了人机交互的沉浸感，使 AI 不再只是冷冰冰的对话框，而是具有“生命感”的虚拟伴侣 。

在本研究中，我们提出了 AIVA——一个将多模态情感感知与大语言模型（LLMs）相结合的框架，旨在实现情感智能的人机交互 。通过整合多模态情感感知网络（MSPN）、情感感知提示工程（EPE）以及通过文本转语音（TTS）和动画虚拟形象实现的表现力输出，AIVA 增强了交互的情感投入度和自然感 。我们的大量实验表明，AIVA 在情感分类、情感对齐和多模态交互方面均优于现有方法，为虚拟伴侣及其他以人为中心的人工智能应用奠定了坚实的基础 。
