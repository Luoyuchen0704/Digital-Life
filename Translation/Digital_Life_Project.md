# Title
数字生命项目：具备社交智能的自主3D角色

# Abstract
在这项工作中，我们提出了数字生命项目（Digital Life Project），这是一个利用语言作为通用媒介来构建自主3D角色的框架 。这些角色能够参与社交互动，并能通过肢体动作进行表达，从而在数字环境中模拟生命 。
我们的框架由两个核心组件组成：
SocioMind：一个精心设计的“数字大脑”，它通过系统的少样本示例（few-shot exemplars）对个性进行建模，结合了基于心理学原理的反思过程，并通过发起对话话题来模拟角色的自主性 。
MoMat-MoGen：一种文本驱动的动作合成范式，用于控制角色的数字身体 。它将行业成熟的**动作匹配（motion matching）技术（确保动作质量）与前沿的动作生成（motion generation）**进展（确保多样性）相结合 。

大量的实验表明，该框架的每个模块在其各自领域都达到了当前最佳（state-of-the-art）的性能 。综合而言，这些模块使虚拟角色能够自主地发起并维持对话，同时演化其社会心理状态 。与此同时，这些角色能够执行与上下文相关的肢体动作 。此外，**动作字幕（motion captioning）**模块进一步允许虚拟角色识别并妥善响应人类玩家的行为 。

# Introduction
最近，大型语言模型 (LLMs) 的飞速发展改变了人机交互的格局，催化了各领域创新应用的涌现 。值得注意的是，许多曾经遥不可及的幻想正逐渐变为现实 。在本研究中，我们采用了近期科幻大片《流浪地球2》中设想的术语——数字生命项目 (Digital Life Project, DLP)，来界定我们的努力方向 。

究竟什么是“数字生命”？ 从心理学角度看，人类是由内在心理过程（心智，如思想）和外在行为组成的 。基于这一观点，我们的目标是利用 LLM 的强大能力来构建虚拟 3D 角色，使其能够模拟全方位的人类心理过程，并结合生成的 3D 身体动作进行多样化的互动 。

最近，Park 等人提出了“生成式智能体” (Generative Agents) ，推动了能够模拟类人行为的 AI 智能体的发展 。尽管取得了令人鼓舞的进展，但这项先驱性工作建立在许多交互简化的基础上：智能体仅以像素化的 2D 图形呈现 。Co-LLM-Agents 旨在构建协作式具身 AI 并包含 3D 智能体 。然而，这些 3D 智能体仍受限于极少数的动作集，且未展现出社交能力 。

因此，现有研究忽视了复杂的人类肢体语言的重要性，而肢体语言传达了关键的信息量 。此外，目前的社交智能 (Social Intelligence) 建模也存在显著缺陷 。这一维度对于创建那些不仅能模仿人类动作，还能拥有类人思考和情感反应、甚至具备培养长期关系能力的角色至关重要 。

为了实现 DLP 的愿景，我们引入了一个由两个核心组件组成的框架 ：
首先是 SocioMind：一个精心设计的“数字大脑”，其设计植根于严谨应用的心理学原理 。
利用 LLM 的涌现能力，该大脑生成高层指令并规划角色的行为 。
值得注意的是，SocioMind 引入了来自心理测试的少样本示例来形成个性建模的指导指令 。
它在记忆反思过程中运用了社会认知心理学理论，并设计了角色间的协商机制以推动故事情节的发展 。
其次是“数字身体”：
引入了 MoMat-MoGen 范式来解决交互式动作合成问题 。

该范式利用了动作匹配 (Motion Matching) 与动作生成 (Motion Generation) 的互补特性 。
其中，动作匹配是现代工业级角色动画的基础技术，通过从数据库检索高质量动作剪辑来确保动作质量 。
而动作生成则是近期迅速兴起的研究方向，能够卓越地产生多样化的人类动作 。
实验结果表明，SocioMind 和 MoMat-MoGen 在各自领域均优于现有技术 。
具体而言，SocioMind 在角色行为与心理状态（如个性与关系）之间展现了出色的对齐性 ；
MoMat-MoGen 则在动作质量与多样性之间实现了平衡 。
配备这两个模块后，我们进一步展示了 DLP 的可控性：手动编辑角色属性可以产生语义准确且美学写实的交互动作 。

此外，我们还通过开发动作字幕 (Motion Captioning) 模块探索了人机交互，该模块能将单目人类视频转化为动作描述，从而使虚拟角色能够理解并妥善响应人类玩家 。

简而言之，我们的贡献包括：DLP：一个构建具备社交特质的自主 3D 角色的框架 。SocioMind：一个可控的、基于心理学的“大脑”，支持短期交互沟通和长期社交演化 。MoMat-MoGen：一个通过协同动作匹配与动作生成来合成高质量、多样化交互动作的“身体” 。


# 1. 相关工作 (Related Works)
## 2.1. 动作合成 (Motion Synthesis)
动作匹配（Motion matching）在工业界被广泛用于生成持久且高质量的动作 。经典的动作匹配 会检索与当前姿态和目标轨迹最匹配的片段 。深度学习动作匹配 则采用自回归神经网络，根据给定的控制信号预测下一个动作状态 。Story-to-motion 进一步通过大型语言模型（LLM）引入了语义控制，并利用 Transformer 模型增强了动作转换 。

最近，在文本驱动的动作生成领域，动作生成模型取得了重大进展 。早期工作旨在为自然语言和动作序列建立统一的隐空间 。Guo 等人 、TM2T 以及 T2M-GPT 采用了自回归方案来生成长动作序列 。基于扩散（Diffusion）的生成模型在文本转动作（text-to-motion）任务的领先基准测试中展现了卓越的性能 。MotionDiffuse 、MDM 和 FLAME 代表了将扩散模型应用于文本驱动动作生成领域的早期尝试 。随后的模型如 MLD 、ReMoDiffuse 、Fg-T2M 和 PhysDiff 进一步推进了这一理念，实现了更好的文本-动作一致性、动作质量和物理合理性 。

## 2.2. LLM 智能体 (LLM Agents)
凭借大型语言模型（LLM）在推理、规划和学习方面的涌现能力 ，LLM 迅速经历了三个进化阶段：独立的原始 LLM、直接通过文本与环境交互的语言智能体 ，以及具有内部认知结构的认知语言智能体 。在认知语言智能体的主要框架下，系统设计取决于预期的应用和目标：例如针对游戏智能体的奖励系统 、针对工具智能体的 API 调用链等 。

此外，LLM 中类人行为的出现促使研究人员调查 LLM 中可控的心理行为，例如稳定的个性 ，以及在政治科学 和社会心理学中的人类模拟 。最近，Social Simulacra 和 S3 构建了在互联网社区空间具有自主发帖和转发技能的智能体系统 。Generative Agents 则通过在 2D 沙盒游戏空间内的日常计划和简短交流，促进了社交关系的形成和信息的扩散 。

# 3. 方法论 (Methodology)
## 3.1. 语言作为通用媒介 (Text as the Universal Medium)
我们定义了行为 (behavior)，这是一种类似字典结构的文本消息，用于在各模块间传递关键信息 。例如，<speech>Hello! <motion>waves right hand <place>table 包含了由尖括号封装的预设键值，后跟相应的自然语言描述 。这种行为描述既能被大型语言模型（LLM）理解，也能被正则表达式解析器处理 。在本研究中，我们重点关注 <motion> 标签，但也讨论了其他标签：<place> 触发导航和基本状态转移（如坐下），<speech> 则可用于面部控制 。

## 3.2. 主动-被动机制 (Active-Passive Mechanism)
人类交互中存在内在的顺序性 。例如，“握手”看似是两个主体同步进行的，但通常由一个人先伸手启动，而另一方的反应在社交礼仪上是高度可预测的 。基于此观察，我们设计了主动-被动机制（如图 2 所示）：主动角色 (Active character)：负责生成行为，并为交互双方合成一对动作（主动动作和被动动作）。
被动角色 (Passive character)：接收被动行为和动作。被动角色保留裁量权，只有当其“大脑”批准该行为时，才会执行相应的被动动作 。随着交互的进行，两者的“主动”和“被动”角色会不断互换 。

## 3.3. 交互式动作合成 (Interactive Motion Synthesis)
生成的动作需满足两个要求：高准确性（确保自然交互，如握手时的接触）和多样性（适应不同情节） 。由于单一的动作匹配或动作生成方法难以同时满足两者，我们提出了 MoMat-MoGen 。
### 3.3.1 动作匹配确保高质量动作先验 (Motion Matching for High-Quality Motion Prior)
我们扩展了文本动作匹配技术以适应双人交互场景 。目标是找到一对既符合查询文本和轨迹，又与前序动作保持姿态连贯的动作剪辑 。
* 语义控制：使用预训练的句子编码器提取文本嵌入，通过余弦相似度筛选候选动作 。
* 运动学约束：利用关节运动学特征（如髋部位置、朝向、关节速度等）确保轨迹对齐和动作连贯 。
* 空间对齐：考虑两个角色的相对位置和朝向，以减少长距离移动带来的融合伪影 。
* 动作融合：使用神经动作融合模型生成平滑的过渡动作 。
## 3.3.2 动作生成提升多样性 (Motion Generation for Diversity)
MoMat-MoGen 在结构上参考了 ReMoDiffuse，但针对交互建模进行了改进 ：
1. DSMA 模块：设计了双路语义调制注意力模块 (Dual-path Semantic-Modulated Attention)，用于建模两人之间的交互 。交互约束：在推理阶段，从参考动作中提取交互信息作为去噪过程的约束，使交互更加自然 。扩散模型 (Motion Diffusion Model)：在反向去噪过程中，结合文本提示、动作匹配结果和时间步，估计出最终的运动序列对 。

## 3.4. 人类心理的可控模拟 (Controllable Emulation of Human Psychology)
我们提出了 SocioMind，一个旨在实现社会智能的文本核心认知框架 。短期交互沟通：受内部心理状态（大五人格、动机、信念、信任、亲密度等）的影响，SocioMind 根据上下文输出行为来操控 3D 角色 。人格指令 (Persona instructions)：通过对公开心理测试（如 IPIP）进行逆向工程，构建“特征-行为”数据库，作为少样本示例增强 LLM 的推理能力 。长期社交演化：心理反思 (Psychological Reflection)：智能体在每个“剧集 (episode)”结束时，反思情感、总结事件与想法，并根据社交心理学理论更新其动机和关系状态 。话题提案规划 (Planning with Topic Proposal)：智能体根据记忆和设定独立提议新话题，双方协商选定下一集的主题和背景，确保故事情节的连贯演进 。

## 3.5. 动作字幕 (Motion Captioning)
我们设计了一个动作转文本的翻译模块，将单目视频捕获的人类动作转化为文本描述 。该模块通过检索增强的动作编码器提取特征，并与预训练语言模型（MPT-1B）对齐，使虚拟角色能够“看懂”并响应人类玩家的动作（如击拳） 。

# 4. 实验 (Experiments) 
据我们所知，“数字生命项目”（Digital Life Project）是首个能够让具备 3D 肢体关节的自主社交角色运行的综合性框架 。因此，我们将该项目拆分为多个模块，以便与现有技术进行公平比较 。

## 4.1. 数据集 (Datasets) 
DLP-MoCap：为了建立高质量的语义交互动作数据集，我们使用了由 30 台摄像机组成的光学动捕系统捕捉身体动作，并使用基于惯性传感器的动捕手套跟踪手部动作 。我们聘请了四位专业演员进行更具表现力的动作表演 。

数据规模：该数据集包含基础动作（22%）、双人交互动作（56%）和短剧表演（22%），累计生成了超过 9 小时、帧率为 30 FPS 的人类动作数据，以及超过 4000 个“文本-动作”对 。

其他数据集：除了 DLP-MoCap，我们还在 InterHuman（交互动作数据集）、KIT-ML 和 HumanML3D（文本-动作数据集）上评估了相关模块 。

## B. DLP 的额外细节 (Additional Details of DLP) 
### B.1. 场景 (Scene) 
我们的目标是构建能在 3D 场景中通过 3D 身体动作进行表达的 3D 角色 。在工作中，我们设计的场景实现了多种相对交互姿态的组合：
1. 中心区或书架：用于双方均处于站立状态的情况（站立-站立交互）。
2. 沙发：用于并排坐着的交互 。
3. 餐桌：用于面对面坐着的交互 。
4. 电脑桌：用于站立-坐姿交互 。

对于每件可互动的家具，我们设计了带有位置和朝向的点位，以引导导航和人机交互 。需要注意的是，本研究重点关注人与人之间的交互，并将人与场景的交互大幅简化为基础动作，如“坐在椅子上”或“从沙发上站起来” 。

### B.2. 动作同步 (Movement Synchronization) 
将单人动作合成扩展到交互式动作合成的一个重大挑战是多角色协作 。虽然我们的“主动-被动机制”通过从精选动作库检索确保了交互的自然对齐，但当角色在场景中从一处移动到另一处时，仍需解决动作长度可能不匹配的问题 。如图 8 所示，每次交互（交互动作对）需要四个步骤来同步角色的动作 ：
* 表现 (Behave)：主动角色通过 SocioMind 生成主动行为，并利用 MoMat-MoGen 合成主动和被动动作 。被动动作传递给被动角色 。
* 移动 (Move)：多智能体路径规划器为两名角色计算无碰撞轨迹 。如果涉及位置变化，则使用动作匹配来合成步行路径 。
* 对齐 (Align)：由于起始位置不同，角色的移动通常需要不同数量的帧 。因此需进行对齐：角色相互沟通其轨迹，轨迹较短的角色会在其轨迹末尾插入填充动作（闲置动作） 。
* 合成 (Synthesize)：将移动动作与交互动作串联，并应用动作融合以确保动作剪辑间的平滑过渡 。

### B.3. 重定向 (Retargeting)
MoMat-MoGen 模块生成的动作采用 SMPL-X 格式 。为了更好地在沉浸式模拟场景中展示社交智能体之间的物理和心理交互，我们使用了两个绑定好的角色：男性 Zhixu 和女性 Xiaotao 。
* 工具：在 Blender 中使用广泛使用的重定向工具 Auto-Rig Pro [84] 将合成动作重定向到目标化身 。配置：手动配置骨骼映射以达到最佳效果 。
* 优化：缩放目标化身使其与 SMPL-X 模型高度一致，以避免明显的“滑步”现象，并保持交互时的身体接触 。

### B.4. 动作匹配 (Motion Matching) 
我们的动作匹配过程分为两步 ：
1. 语义提取：利用预训练 LLM 提取查询文本的文本嵌入 $f_t \in \mathbb{R}^{1024}$ 。
2. 运动学匹配：合并运动学特征（如轨迹、朝向、关节旋转、髋部高度和相对位置）以增强连贯性和轨迹对齐 。
3. 距离度量：除朝向相似度采用余弦距离外，其余均采用欧几里得距离 。归一化：相似度通过 Z-Score 归一化后加权求和 。

### B.5. 动作生成 (Motion Generation) 
DSMA 架构细节：我们的双路语义调制注意力 (DSMA) 模块建立在 ReMoDiffuse 的 SMA 模块之上 。主要区别在于引入了动作交互 。为了获得精炼特征 $\hat{f}_x$，我们整合了四类特征源 ：同一序列的独立动作特征 $f_x$ 。来自伙伴序列的孪生动作特征 $f_y$ 。来自给定描述 $P$ 的文本特征 $p_x$ 。来自动作匹配结果 $M_x$ 的参考特征 $r_x$ 。

# 数字生活项目2 (DLP2) 
提出了一种开源的实时框架，它通过富有表现力的3D虚拟形象，将大型语言模型 (LLM) 栩栩如生地呈现在用户面前。用户可以通过语音进行自然对话，而角色则可按需做出响应，在浏览器中直接呈现统一的音频、全身动画和物理模拟效果。其核心包括：(1) 一个由大型和小型LLM组成的智能体编排系统，用于控制角色行为，并由一个记忆系统提供支持，该系统跟踪情绪状态和不断变化的关系，从而实现与上下文相关的反应；(2) 一个混合实时流水线，用于分割较长的LLM响应，执行并行动作检索和音频-动作同步，并通过自定义的Protocol Buffers结构高效地传输数据，以实现语音、动作和表情的低延迟播放；(3) 强大的用户中断处理、自适应缓冲和容错机制。角色的外观（3D模型）和个性（角色提示）均可完全自定义，并且可以轻松适配任何LLM或文本转语音 (TTS) 服务。 DLP2 展示了如何将 LLM 体现在响应式 3D 角色中，为在网络上进行实时、情感自适应的数字交互提供了一个实用的蓝图。
